# Change Log

## 0.1.23

- 取消“代码备注”与系统资源管理器之间的目录展开单向同步，两棵树各自维护展开状态，避免产生双向同步的错误预期。

## 0.1.22

- “代码备注”资源树跟随活动编辑器，自动展开并选中当前文件。
- 手动展开“代码备注”目录时，单向在系统资源管理器中定位该目录。
- 移除冗余的 `N` badge，保留备注主题色和完整 tooltip。

## 0.1.21

- JetBrains 恢复 `Shift+Option+R` / `Shift+Alt+R` 设置备注样式快捷键；发生冲突时由用户在 Keymap 中自行调整。

## 0.1.20

- 样式实时预览改为只更新当前节点，不再刷新整棵“代码备注”资源树。
- JetBrains 取消与“重新运行测试”冲突的默认样式快捷键；仍可在 Keymap 中自行绑定。

## 0.1.19

- 统一“代码备注”资源树中的编辑、设置样式和清除备注菜单，所有可编辑节点保持一致。

## 0.1.18

- 修复“代码备注”资源树中部分节点缺少“设置备注样式”入口的问题。
- 完善 Marketplace 产品介绍、截图、关键词与项目链接。
- 新增默认快捷键：编辑文字备注使用 `Option+R`（macOS）/ `Alt+R`（Windows），设置备注样式使用 `Shift+Option+R`（macOS）/ `Shift+Alt+R`（Windows）。

## 0.1.17

- 恢复 VS Code 原生“代码备注”资源树，并保留原生主题、键盘操作和右键菜单。
- 增加默认、信息、成功、警告、危险五种备注样式和实时预览。
- 备注颜色仅显示在“代码备注”视图，不改变系统资源管理器的文件颜色。
- 增加系统资源管理器与“代码备注”之间的双向定位。
- 修复 Git 忽略目录覆盖备注颜色、快捷键目标错误等问题。
- 简化样式选择器并完善中英文界面。

## 0.1.10

- 首次发布到 Visual Studio Marketplace。
- 支持文件与目录备注、搜索、路径迁移和 `.codebase-notes.json` 共享配置。
- 支持 VS Code、TRAE 与 JetBrains IDE 双端协作。
